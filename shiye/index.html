创业思路梳理，过滤掉不靠谱的想法

第一关：它是以人为本的 必须是个你自己的问题或者痛点，自己就是核心用户，而非猜测用户的喜好、痛点和可能的解决方法，这些痛点必须足够“痛”，否则可能只有你一个用户。有没有其他的方法可以达到？真的需要用这个方法才好吗？ 第二关：它具有受众 先找到一百个愿意当你的内测的珍贵的种子用户，帮你完善你的核心想法，改善用户体验，并帮你在正式推广前更正一些错误。做一个简单的网站或一个app原型（不必真的能用，能点就行）然后往上面导流。找些人注册下载并且感受一下用户体验，问问他们什么地方是他们喜欢的，以及他们认为这个app是如何解决了现实中的问题的。 第三关：它简洁而完整 30秒必须简洁得让任何人都能秒懂你能做什么。同时需要提供某些“完整”的服务，比如按一个键就能打到出租车并且顺利支付车钱。把项目给随便什么人看但别做过多解释如果他们不太明白就需要继续努力。 第四关：它是独特的 人们看到你的app而被惊艳到，那它应该就足够独特了。 第五关：它很小 今天很小未来很大的“影子市场”。做一份商业计划书给创投圈内的朋友们看。如果大家都觉得这个想法有点“小”，那它可能反而倒还不错。 最后：别浪费时间去主动找融资。 老想着融资会让你“出轨”——不能一心扑在产品上。如果你做的app很吸引人，你会极快地获得很多帮助。所以别急着动手，先彻底检查一下你的想法，如果它通过了所有测试，那就集中精力在产品上，努力吸引用户。吸引力胜过一切。有了它，万事大吉。

心得 网络编程发展趋势 1、网页即未来，网页相对于本地软件更有未来优势 网页更省资源，更安全（无病毒），更快 电脑，手机，平台 修改之后即时生效，不用升级，

2、网页编程发展一 服务器通过对数据库数据的解析和计算把网页代码解析好之后返回给浏览器 网速慢，网页慢，电脑也慢，网页丑，使用不方便，（特别是图片最慢）

3、网页编程发展二 用户只需要少量变换页面中的内容， 比如用户名是否被别人注册了，不用提交了之后才告诉我，那么多东西都白填了 javascript，Ajax技术 PHP + HTML5 + CSS3 + JAVASCIRPT

4、网页编程发展三 用户需要更加迅速的页面展现和更新 比如刷微博，不断的往下拉，还要可以看天气预报 JQuery，xml，json，

5、网页编程发展四 新手拿到了一手臭牌 javascript最开始极其简陋的设计 不同的用户喜欢用不同的浏览器 臭名昭著的IE6浏览器 前端技术员陷入了苦难的深渊

6、网页编程发展五 不断提升的用户体验需求和落后的前端技术标准的矛盾 ES5(ECMAScript5)，ES6(ECMAScript6) 2016第一个季度，，各大浏览器遵循的标准 javascript(动态类型),coffescript,typescript(更专业高效，静态类型好把握，强类型，可定义扩展接口，继承，模块化，装饰器), jquery，bootstrap，angularjs，react 物联网

学习后端经验

7、网页编程发展六 网页 VS 本地软件 SPA单页Web应用（single page web application，SPA），就是只有一张Web页面的应用，是加载单个HTML 页面并在用户与应用程序交互时动态更新该页面的Web应用程序。 不需要联网你就需要的软件以后逐渐会被系统自带 需要联网才能用的软件，以后都会编程网页形式 小程序

8、网页编程发展七 世界大同 每个人都可以在手机上办公，个人服务器随身携带，交换信息 网页控制家中电器，自动下载好
